## Context

### Commit Message Conventions

**Guidelines:**

- Keep commit messages straightforward and to the point
- Only describe the code changes made
- Do not include extraneous details or meta-commentary

**What NOT to include:**

- Never mention Claude, Anthropic, AI, or "Generated with"
- No tool attribution in commit messages
- No meta-commentary about the process

**Format:**

```
<type>: <brief description>

<optional body explaining why, not what>
```

**Types:**

- `feat`: New feature
- `fix`: Bug fix
- `refactor`: Code change that neither fixes a bug nor adds a feature
- `docs`: Documentation changes
- `test`: Adding or updating tests
- `chore`: Maintenance tasks
