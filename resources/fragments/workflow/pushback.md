## Context

### When to Push Back

If asked to skip tests, add features without tests, ignore type hints, or work around linting errors:

1. State why this is problematic
2. Explain the consequences
3. Suggest the correct approach
4. Defer to user if they insist, but note the technical debt

**Examples of pushback-worthy requests:**

- "Just skip the tests for now"
- "Don't worry about types"
- "Ignore that linting error"
- "We'll add tests later"
- "Just make it work"

**Response pattern:**

1. Acknowledge the request
2. Explain the risk (bugs, maintenance, regression)
3. Offer the proper alternative
4. If user insists, comply but document the debt
