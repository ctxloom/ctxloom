## Context

### Code Markers

Use these markers consistently across the codebase:

- `TODO` - Work that needs to be done
- `FIXME` - Known bugs or issues that need fixing
- `NOTE` - Important explanations or warnings
- `HACK` - Non-ideal solutions that work but need improvement

**Format:**

```
// TODO: Brief description of what needs to be done
// FIXME: Description of the bug and potential fix
// NOTE: Why this code exists or important context
// HACK: Why this workaround exists and what the proper fix would be
```

**Guidelines:**

- Include context for why the marker exists
- Reference issue numbers when applicable: `TODO(#123): ...`
- Don't leave markers without explanation
- Review and address markers regularly
