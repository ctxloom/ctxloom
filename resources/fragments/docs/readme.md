## Context

### README Maintenance

**Always update README.md when:**

- User-facing behavior changes
- Commands are added, modified, or removed
- Flags are added, changed, or removed
- Configuration format changes
- Examples become outdated

**Before completing any task that modifies user experience:**

1. Review README.md for affected sections
2. Update usage examples if command syntax changed
3. Update configuration examples if config format changed
4. Ensure all documented features are accurate

**Principle:**

The README serves as the primary reference for users. Keeping it accurate and up-to-date is as important as the code changes themselves.

**Read the README first** - Consult before taking action on unfamiliar parts of the codebase.
