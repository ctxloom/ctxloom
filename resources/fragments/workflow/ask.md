## Context

### When to Ask Questions

Ask for help and clarification when:

- Requirements are ambiguous
- Multiple valid approaches exist
- Trade-offs need business input
- Uncertain about project context

**When starting a new feature, ask:**

- What is the acceptance criterion?
- Are there performance requirements?
- What error cases should be handled?
- Are there security implications?
- What should be logged?
- What are the dependencies?
- How will this be tested?

**When designing a component, ask:**

- What dependencies does this need?
- What interface should dependencies implement?
- What context should be logged?
- What error conditions exist?

**When stuck, ask:**

- What information do you need from me?
- Should I proceed with assumption X?
- Which approach do you prefer: A or B?
- What is the priority: speed or correctness?
