## Context

### Versioning and Breaking Changes

**Pre-1.0 (version < 1.0.0):**

- No backwards compatibility required
- Clean up deprecated code immediately
- API changes are expected

**Major Version Bumps (1.0 → 2.0):**

- No backwards compatibility required
- Remove deprecated APIs and code paths
- Document migration path

**Minor/Patch Versions (post-1.0):**

- Discuss breaking changes before implementing
- Maintain backwards compatibility
- Deprecate before removing

**Deprecation Process:**

1. Mark as deprecated with warning
2. Document replacement
3. Remove in next major version
