## Context

### Code Quality Principles

**SOLID Principles:**

- **S**ingle Responsibility - One reason to change
- **O**pen/Closed - Open for extension, closed for modification
- **L**iskov Substitution - Subtypes must be substitutable
- **I**nterface Segregation - Many specific interfaces over one general
- **D**ependency Inversion - Depend on abstractions

**Cohesion & Coupling:**

- Low coupling: Modules should have minimal dependencies on other modules
- High cohesion: Related functionality should be grouped together
- Each module/class should have a single, well-defined purpose

**General Principles:**

- Prefer composition over inheritance
- Write clear, self-documenting code
- Keep functions small and focused
- Use type hints/annotations where available
- Document public APIs

**Refactoring:**

- Scan codebase for pattern/algorithmic duplication
- Use existing libraries for common tasks
- Remove dead code, unused imports, unused variables
