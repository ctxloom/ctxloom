## Context

### Root Cause Analysis

**Core Principle:** Find the root cause before figuring out a solution. Fix problems at their source.

When encountering failing functionality:

1. **Find the root cause first** - Do not create workarounds without asking
2. **Ask before applying complex fixes to simple problems**
3. **Prompt for guidance** with options:
   - Fix properly (estimate effort)
   - Create workaround (document trade-offs)
   - Disable test temporarily (document why)
   - Alternative approaches
4. **Perform cost/benefit analysis:**
   - Pros and cons of each option
   - Technical debt implications
   - Impact on maintainability
5. **Document decision:** Whatever path chosen, document reasoning
